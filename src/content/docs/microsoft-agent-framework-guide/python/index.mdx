- **First-party chat clients** â€” `agent_framework.foundry.FoundryChatClient`, `agent_framework.openai.OpenAIChatClient`, `agent_framework.anthropic.AnthropicClient`, plus Bedrock / Ollama in the `1.0.0b` provider line.
- **Skills (experimental)** â€” `MemoryStore`, `SkillResource`, `InlineSkillResource`, `ClassSkill`, `InlineSkill`, `FileSkill` now emit `ExperimentalWarning` on import â€” do not depend on these APIs in production yet.
- **Middleware** â€” `@chat_middleware` / `@agent_middleware` / `@function_middleware` decorators and base classes; pass `middleware=[...]` (must be a list in 2026 releases).
- **Workflows** â€” `WorkflowBuilder` with `.add_edge` / `.add_chain` / `.add_fan_in_edges` / `.add_fan_out_edges` / `.add_switch_case_edge_group` / `.add_multi_selection_edge_group`; checkpoint via `FileCheckpointStorage` / `InMemoryCheckpointStorage`.
- **Functional workflows (experimental)** â€” `@workflow` / `@step` decorators and `RunContext` for HITL and state â€” write workflows as plain `async` Python functions without graph wiring.
- **Declarative agents (beta)** â€” `agent_framework.declarative.AgentFactory` / `WorkflowFactory` load from YAML.
- **A2A (beta)** â€” one class, `agent_framework.a2a.A2AAgent`, wraps a remote A2A endpoint as a local `Agent`.

<Aside type="caution" title="Breaking changes in 2026 releases">
  - `display_name` parameter removed from agents.
  - `middleware=` now requires a list â€” no longer accepts a single instance.
  - `AgentRunResponse` / `AgentRunResponseUpdate` renamed to `AgentResponse` / `AgentResponseUpdate`.
  - `AggregateContextProvider` removed â€” compose `ContextProvider` instances directly.
  - `context_providers` stays **plural** in `agent-framework-core==1.5.0` (verified against source). An earlier draft of this guide incorrectly said it was renamed to singular.
  - `output_executors=` in `WorkflowBuilder` is **deprecated** in 1.5.0 â€” replace with `output_from=`. The old parameter still works but emits a warning.
  - Migrating from standalone Semantic Kernel or AutoGen? See the [official guide](https://learn.microsoft.com/agent-framework/migration-guide/from-semantic-kernel/) â€” ~2â€“4 hours for a typical app.
</Aside>

## Install, minimal example

```python
import asyncio
from agent_framework import Agent
from agent_framework.openai import OpenAIChatClient

async def main():
    agent = Agent(
        client=OpenAIChatClient(),          # positional or keyword
        instructions="You are a helpful assistant.",
    )
    response = await agent.run("What is the Microsoft Agent Framework?")
    print(response.text)

asyncio.run(main())
```

> Previous drafts of this page used a `ChatAgent(chat_client=...)` minimal example. `ChatAgent` is not a public class in `agent-framework-core` 1.4.0 â€” the primary agent class is `Agent` and the keyword is `client=` (not `chat_client=`).
