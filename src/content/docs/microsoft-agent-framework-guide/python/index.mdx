**Unified agent SDK that merges Semantic Kernel and AutoGen** into one Pythonic library. Build single agents, multi-agent orchestration, cross-framework collaboration (A2A), and declarative workflows against the same primitives.

<CardGrid>
  <Card title="Install" icon="rocket">
    ```bash
    pip install agent-framework
    ```
  </Card>
  <Card title="Version" icon="information">
    Core **1.8.1** stable ¬∑ `agent-framework-foundry` and `agent-framework-openai` **1.8.0 stable** ¬∑ June 2026 ¬∑ Python 3.10 ‚Äì 3.13
  </Card>
  <Card title="Best for" icon="puzzle">
    Azure-native agents, multi-agent orchestration via `WorkflowBuilder`, cross-framework A2A, declarative YAML agents.
  </Card>
</CardGrid>

<Aside type="caution" title="Earlier drafts had fabricated APIs">
  Pages under this guide previously documented `agent_framework.graphs.AgentGraph`, `agent_framework.a2a.A2AProtocolAdapter`, `AzureAIAgentClient`, `ChatAgent`, `ai_function`, and other names that **do not exist**. The real primary agent class in Python is `Agent`; the Azure chat client is `FoundryChatClient` from `agent_framework.foundry`. Pages were rewritten in April 2026 after introspecting `agent-framework-core==1.1.0` and re-verified in May 2026 against `1.5.0`. See [Errata in the comprehensive guide](/microsoft-agent-framework-guide/python/microsoft_agent_framework_python_comprehensive_guide/) for the running list.
</Aside>

<Aside type="tip" title="azure-ai-agents integration add-on ‚Äî class reference (6 volumes)">
  The `azure-ai-agents` package is an optional Microsoft add-on for the Azure AI Agents service. It is **not** a replacement for `agent-framework` ‚Äî use it alongside the framework when you need direct access to Azure AI Agents service resources. Source-verified class deep dives covering **80+ classes/patterns** across 6 volumes are available in the Integration Reference section below.
</Aside>

- **First-party chat clients** ‚Äî `agent_framework.foundry.FoundryChatClient`, `agent_framework.openai.OpenAIChatClient`, `agent_framework.anthropic.AnthropicClient`, plus Bedrock / Ollama in the `1.0.0b` provider line.
- **Skills (experimental)** ‚Äî `MemoryStore`, `SkillResource`, `InlineSkillResource`, `ClassSkill`, `InlineSkill`, `FileSkill` now emit `ExperimentalWarning` on import ‚Äî do not depend on these APIs in production yet.
- **Middleware** ‚Äî `@chat_middleware` / `@agent_middleware` / `@function_middleware` decorators and base classes; pass `middleware=[...]` (must be a list in 2026 releases).
- **Workflows** ‚Äî `WorkflowBuilder` with `.add_edge` / `.add_chain` / `.add_fan_in_edges` / `.add_fan_out_edges` / `.add_switch_case_edge_group` / `.add_multi_selection_edge_group`; checkpoint via `FileCheckpointStorage` / `InMemoryCheckpointStorage`.
- **Functional workflows (experimental)** ‚Äî `@workflow` / `@step` decorators and `RunContext` for HITL and state ‚Äî write workflows as plain `async` Python functions without graph wiring.
- **Declarative agents (beta)** ‚Äî `agent_framework.declarative.AgentFactory` / `WorkflowFactory` load from YAML.
- **A2A (beta)** ‚Äî one class, `agent_framework.a2a.A2AAgent`, wraps a remote A2A endpoint as a local `Agent`.

<Aside type="caution" title="Breaking changes in 2026 releases">
  - `display_name` parameter removed from agents.
  - `middleware=` now requires a list ‚Äî no longer accepts a single instance.
  - `AgentRunResponse` / `AgentRunResponseUpdate` renamed to `AgentResponse` / `AgentResponseUpdate`.
  - `AggregateContextProvider` removed ‚Äî compose `ContextProvider` instances directly.
  - `context_providers` stays **plural** in `agent-framework-core==1.5.0` (verified against source). An earlier draft of this guide incorrectly said it was renamed to singular.
  - `output_executors=` in `WorkflowBuilder` is **deprecated** in 1.5.0 ‚Äî replace with `output_from=`. The old parameter still works but emits a warning.
  - Migrating from standalone Semantic Kernel or AutoGen? See the [official guide](https://learn.microsoft.com/agent-framework/migration-guide/from-semantic-kernel/) ‚Äî ~2‚Äì4 hours for a typical app.
</Aside>

## Install, minimal example

```python
import asyncio
from agent_framework import Agent
from agent_framework.openai import OpenAIChatClient

async def main():
    agent = Agent(
        client=OpenAIChatClient(),          # positional or keyword
        instructions="You are a helpful assistant.",
    )
    response = await agent.run("What is the Microsoft Agent Framework?")
    print(response.text)

asyncio.run(main())
```
