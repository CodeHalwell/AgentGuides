<Aside type="danger" title="Package name correction — action required">
  Earlier versions of this guide documented a package called `agent-framework` (import: `agent_framework`) with classes like `Agent(client=..., instructions=...)` and `FoundryChatClient`. **That package does not match the real Azure AI Agents SDK.**

  The **correct** package is **`azure-ai-agents`** (import: `azure.ai.agents`) with the main class **`AgentsClient`**. All new content on this page and in the new SDK deep-dives document uses the correct package. ## What's shipped (May 2026 introspection)

- **Stable core APIs** — `agent-framework-core 1.6.0`. `agent-framework-foundry` and `agent-framework-openai` updated to **1.6.0** in this release. All 23 core symbols (`Agent`, `BaseAgent`, `RawAgent`, `AgentContext`, `AgentSession`, `Runner`, `FunctionTool`, `AgentMiddleware`, `ChatMiddleware`, `FunctionMiddleware`, `WorkflowBuilder`, `FunctionalWorkflow`, `RunContext`, `CheckpointStorage`, `FileCheckpointStorage`, `MemoryContextProvider`, `MemoryStore`, `SkillsProvider`, `InlineSkill`, `FileSkill`, `MCPStdioTool`, `MCPStreamableHTTPTool`, `MCPWebsocketTool`) verified against installed `agent-framework-core==1.6.0` (`.routine-envs/check-0522-ms-agent`); all PASS. 242 total public symbols.
- **First-party chat clients** — `agent_framework.foundry.FoundryChatClient`, `agent_framework.openai.OpenAIChatClient`, `agent_framework.anthropic.AnthropicClient`, plus Bedrock / Ollama in the `1.0.0b` provider line.
- **Skills (experimental)** — `MemoryStore`, `SkillResource`, `InlineSkillResource`, `ClassSkill`, `InlineSkill`, `FileSkill` now emit `ExperimentalWarning` on import — do not depend on these APIs in production yet.
- **Middleware** — `@chat_middleware` / `@agent_middleware` / `@function_middleware` decorators and base classes; pass `middleware=[...]` (must be a list in 2026 releases).
- **Workflows** — `WorkflowBuilder` with `.add_edge` / `.add_chain` / `.add_fan_in_edges` / `.add_fan_out_edges` / `.add_switch_case_edge_group` / `.add_multi_selection_edge_group`; checkpoint via `FileCheckpointStorage` / `InMemoryCheckpointStorage`.
- **Functional workflows (experimental)** — `@workflow` / `@step` decorators and `RunContext` for HITL and state — write workflows as plain `async` Python functions without graph wiring.
- **Declarative agents (beta)** — `agent_framework.declarative.AgentFactory` / `WorkflowFactory` load from YAML.
- **A2A (beta)** — one class, `agent_framework.a2a.A2AAgent`, wraps a remote A2A endpoint as a local `Agent`.

<Aside type="caution" title="Breaking changes in 2026 releases">
  - `display_name` parameter removed from agents.
  - `middleware=` now requires a list — no longer accepts a single instance.
  - `AgentRunResponse` / `AgentRunResponseUpdate` renamed to `AgentResponse` / `AgentResponseUpdate`.
  - `AggregateContextProvider` removed — compose `ContextProvider` instances directly.
  - `context_providers` stays **plural** in `agent-framework-core==1.5.0` (verified against source). An earlier draft of this guide incorrectly said it was renamed to singular.
  - `output_executors=` in `WorkflowBuilder` is **deprecated** in 1.5.0 — replace with `output_from=`. The old parameter still works but emits a warning.
  - Migrating from standalone Semantic Kernel or AutoGen? See the [official guide](https://learn.microsoft.com/agent-framework/migration-guide/from-semantic-kernel/) — ~2–4 hours for a typical app.
</Aside>

## Install, minimal example

```python
import os
from azure.ai.agents import AgentsClient
from azure.ai.agents.models import ListSortOrder, MessageRole
from azure.identity import DefaultAzureCredential

# Endpoint: https://<aiservices-id>.services.ai.azure.com/api/projects/<project-name>
endpoint = os.environ["AZURE_AI_AGENTS_ENDPOINT"]
client = AgentsClient(endpoint=endpoint, credential=DefaultAzureCredential())

# 1. Create a persistent agent (model + instructions stored in Azure)
agent = client.create_agent(
    model="gpt-4o",
    name="HelpfulAssistant",
    instructions="You are a helpful assistant.",
)

# 2. Create a conversation thread
thread = client.threads.create()

# 3. Add a user message
client.messages.create(
    thread_id=thread.id,
    role=MessageRole.USER,
    content="What is the Azure AI Agents SDK?",
)

# 4. Run the agent and poll until done
run = client.runs.create_and_process(thread_id=thread.id, agent_id=agent.id)
print(f"Run status: {run.status}")

# 5. Read the assistant's reply
for message in client.messages.list(thread_id=thread.id, order=ListSortOrder.ASCENDING):
    if message.role == MessageRole.AGENT:
        for content in message.text_messages:
            print(f"Assistant: {content.text.value}")

# 6. Tidy up
client.threads.delete(thread.id)
client.delete_agent(agent.id)
```
