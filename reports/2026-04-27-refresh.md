**Changes applied:**

1. **`HfApiModel` status corrected — docs-vs-source discrepancy.** The guide at line 1005 read `"HfApiModel has been deprecated"` and the code block at line 1014 stated `"# DEPRECATED (still works with shim in v1.24.0):"`. Symbol verification confirms `from smolagents import HfApiModel` raises `ImportError: cannot import name 'HfApiModel'` against installed v1.24.0 — there is no shim. The prose and comment have been updated to state the symbol is **removed** and raises `ImportError`.

   - Source: `smolagents/__init__.py` in `.routine-envs/smol-py-0427/lib/python3.12/site-packages/smolagents/` — `HfApiModel` absent from `__all__` and all namespace exports.

2. **Invalid `max_retries` constructor parameter removed.** The `InferenceClientModel` "full configuration example" included `max_retries=3` as a direct constructor argument. Signature inspection against installed v1.24.0 confirms the constructor parameters are: `model_id`, `provider`, `token`, `timeout`, `client_kwargs`, `custom_role_conversions`, `api_key`, `bill_to`, `base_url`, and `**kwargs`. `max_retries` is not present as a dedicated parameter. When passed via `**kwargs` it is stored in `self.kwargs` and forwarded to generation calls — not used for retry logic. The parameter has been removed from the example; `timeout=120` (the actual default) replaces it.

   - Source: `inspect.signature(InferenceClientModel.__init__)` against installed v1.24.0 — `max_retries` absent.
