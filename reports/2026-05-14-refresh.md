### 2.3 semantic-kernel 1.41.3 → 1.42.0

**Install command:** `uv pip install semantic-kernel==1.42.0 --prerelease=allow` in `.routine-envs/check-0514-sk`  
**Resolved version:** 1.42.0  
**Install date:** 2026-05-14  
**PyPI URL:** https://pypi.org/project/semantic-kernel/1.42.0/  
**Environment:** Python 3.12.3; pydantic 2.13.4 (resolved by semantic-kernel's own constraint `pydantic>=2.0,<2.14`)

**Note on install:** `uv pip install semantic-kernel==1.42.0` (without `--prerelease`) fails because the package requires `azure-ai-agents>=1.2.0b3` — a pre-release version. The `--prerelease=allow` flag is required. This is noted in the risk register.

**Source findings (`.routine-envs/check-0514-sk/lib/python3.12/site-packages/`):**

Minor release from 1.41.3. Key changes observed in installed source metadata:

- `mcp` requirement raised from `>=1.8` to `>=1.26.0`.
- `azure-ai-agents>=1.2.0b3` added as a hard pre-release dependency.
- `typing-extensions>=4.13` now required.

**Docs-vs-source discrepancy — `PydanticDeprecatedSince211`:** Importing `from semantic_kernel.connectors.ai.open_ai import OpenAIChatCompletion` emits `pydantic.warnings.PydanticDeprecatedSince211` at module load time. The warning originates from `AzureAISearchDataSource(AzureChatRequestBase)` at `azure_chat_prompt_execution_settings.py:134`, which overrides `__get_pydantic_core_schema__` — a pattern deprecated in pydantic 2.11 and targeted for removal in pydantic 3.0. This is triggered by pydantic 2.13.4 (installed by semantic-kernel's own dependency resolution). This is an internal semantic-kernel bug; it affects any guide code that imports `OpenAIChatCompletion`. The guide's minimal example and many snippets use this import. The warning does not prevent the guide code from running; it is not an error in `-W default` mode. Flagged in the risk register.

**Changes applied:**

1. `semantic-kernel-guide/python/index.mdx`: version card `v1.41.3 · April 2026` → `v1.42.0 · May 2026`; "What's new" heading updated to v1.42.0; v1.42.0 bullet prepended noting `mcp>=1.26.0`, pre-release dependency, and `PydanticDeprecatedSince211` warning; revision history entry added.
