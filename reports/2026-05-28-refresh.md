### 2. Semantic Kernel (Python) — version discrepancy note (carry-forward)

**Context:** The guide declares v1.42.0 with a note that it requires `--pre` / `--prerelease=allow`. Installing `semantic-kernel` with `uv pip install semantic-kernel --prerelease=allow` resolves to **1.36.0** in this environment.

**Root cause investigation:** `uv pip install 'semantic-kernel==1.42.0' --dry-run` fails with:

```
hint: `azure-ai-agents` was requested with a pre-release marker
(e.g., azure-ai-agents>=1.2.0b3), but pre-releases weren't enabled
```

This confirms that semantic-kernel 1.42.0 exists on PyPI but transitively requires `azure-ai-agents>=1.2.0b3` (a pre-release). When `--prerelease=allow` is applied to `semantic-kernel` itself but not propagated to transitive dependencies, uv resolves back to the highest version compatible with stable `azure-ai-agents` (1.1.0), which is semantic-kernel 1.36.0.

**Resolution:** The guide's instruction (`pip install --pre semantic-kernel`) is technically correct for reaching 1.42.0 — pip's `--pre` flag propagates to all dependencies. The guide accurately documents this constraint. No update to guide content is required.

**Action:** Flag for human review: users on `uv` should use `uv pip install --prerelease=allow semantic-kernel` *and* allow pre-release for azure-ai-agents, or use `pip install --pre semantic-kernel`. The guide's current install snippet uses `pip install --pre` which is correct.
