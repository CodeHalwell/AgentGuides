### 2.7 Semantic Kernel ‚Äî docs-vs-source discrepancy (installation dependency issue)

**Situation:** `semantic-kernel==1.41.2` (latest stable on PyPI) depends on `azure-ai-agents>=1.2.0b3`, which is a pre-release package. A plain `uv pip install --upgrade semantic-kernel` resolves to `1.36.0` because `uv` respects stable-only constraints by default.

**Resolution:** `semantic-kernel==1.41.2` installs correctly with `uv pip install "semantic-kernel==1.41.2" --prerelease=allow`. Verified: `import importlib.metadata; importlib.metadata.version('semantic-kernel') == '1.41.2'` ‚Äî PASS.

**Guide impact:** None ‚Äî the guide and `frameworks.ts` already document `v1.41.2`. No version change is needed.

**Risk flag:** Downstream users following a plain `pip install semantic-kernel` or `uv pip install semantic-kernel` without allowing pre-releases may silently land on 1.36.0 instead of 1.41.2. The install snippet in the guide does not currently note this caveat.

**Open item for human review:** Decide whether to add an install note to the Semantic Kernel guide warning users that `azure-ai-agents` is a pre-release dependency and suggesting `pip install semantic-kernel` (which respects PyPI's pre-release allow for declared dependencies) over uv's default strict mode.
