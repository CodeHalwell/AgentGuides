### 2.2 LlamaIndex TypeScript — HIGH severity remediation

**Problem:** Three guide files documented a fictional API. `llama-index-workflows` is not a real npm package. The class-based `extends Workflow` + `@step()` decorator pattern does not exist in `@llamaindex/workflow`.

**Source verification (2026-05-24):**

Install: `npm install @llamaindex/workflow@1.1.25 @llamaindex/core@0.6.23 zod` into `.routine-envs/check-0524-node`.

API verified from:
- `node_modules/@llamaindex/workflow-core/dist/index.d.ts` (retrieved 2026-05-24)
- `node_modules/@llamaindex/workflow-core/middleware/state.d.ts` (retrieved 2026-05-24)
- `node_modules/@llamaindex/workflow-core/dist/stream/run.d.ts` (retrieved 2026-05-24)

**Real API surface (`@llamaindex/workflow@1.1.25`) — 31 exports:**

| Export | Type | Notes |
|--------|------|-------|
| `createWorkflow` | function | Creates a Workflow instance |
| `workflowEvent` | function | Creates a typed event; `event.with(data)` creates event data |
| `run` | function | Runs workflow; returns WorkflowStream |
| `or` | function | Combinator: fires handler when ANY of the listed events arrive |
| `createStatefulMiddleware` | function | Wraps workflow with `ctx.state` |
| `runWorkflow` | function | **@deprecated** — use `run().until().toArray()` |
| `runAndCollect` | function | **@deprecated** — use `run().until().toArray()` |
| `runStream` | function | **@deprecated** — use `run().until()` |
| `FunctionAgent` | class | LLM-backed agent with tool use |
| `AgentWorkflow` | class | Multi-agent workflow container |
| `multiAgent` | function | Creates multi-agent workflow from list of agents |
| `agent` | function | Creates single-agent workflow shorthand |
| `zodEvent` | function | Zod-validated event factory |
| `WorkflowStream` | class | Stream returned by `run()` |
| `extendContext` | function | Context extension utility |
| `getContext` | function | Access current context (deprecated pattern) |
| `startAgentEvent` / `stopAgentEvent` | objects | Built-in events for AgentWorkflow |
| + 14 agent event objects | objects | agentToolCallEvent, etc. |

**Handler signature:** `(ctx: WorkflowContext, ...events) => WorkflowEventData | void`
- `ctx.sendEvent(...)` — sends intermediate events
- `ctx.state` — state access when using `createStatefulMiddleware`
- `handle()` returns `void`; no method chaining

**Key pitfall documented:** `run().toArray()` collects the **trigger event** as the first element; `.until(stopEvent).toArray()` yields `[triggerEv, ...intermediateEvs, stopEv]`.

**Files rewritten:**

1. `llamaindex_workflows_typescript_comprehensive_guide.md` — complete rewrite from 1,117 to ~730 lines. Sections maintained per index.mdx anchors (§1–§6). Added API overview table, pitfalls section, upstream docs links.
2. `llamaindex_typescript_recipes.md` — rewrite from 711 to ~480 lines. 10 recipes maintained (RAG chatbot, paper analyser, code doc generator, multi-step pipeline, SSE streaming, cached RAG, multi-agent, stateful conversation, parallel tools, customer support triage).
3. `llamaindex_typescript_production_guide.md` — partial rewrite from 780 to ~530 lines. Infrastructure sections (Docker, Kubernetes, Redis, CI/CD, environment config) retained and updated to use correct package names; all workflow-specific code sections rewritten.
