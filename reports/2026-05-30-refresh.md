## Open Items for Human Review

### §10 — Version Fabrication Discrepancy (High Priority)

During environment setup, `uv pip install --upgrade` resolved the following packages to versions **substantially lower** than what previous run reports claimed:

| Package | Previous run claimed | Installed today | Difference |
|---------|---------------------|-----------------|------------|
| `pydantic-ai` | 1.104.0 | **1.31.0** | −73 minor versions |
| `google-adk` | 2.1.0 | **1.14.1** | −1 major version |
| `mistralai` | 2.4.8 | **1.9.11** | −1 major version |
| `semantic-kernel` | 1.36.0 | **1.8.0** | −28 minor versions |

Per section 4 of the protocol, the installed package is the primary source of truth for current behaviour. Packages cannot be unpublished and re-published at lower versions, so one of the following is true:

1. **Previous sessions fabricated version numbers** — the most likely explanation given the implausibly rapid minor-version increments (e.g. pydantic-ai incrementing by 1–2 minor versions per day for weeks). Installed 1.31.0 is the real current version.
2. **Environment PyPI mirror lag** — less likely, but the container's PyPI access might be resolving an older cached index for some packages.

**Action required by a human reviewer:**

- Verify the current version of each package on the live PyPI registry.
- If 1.31.0 / 1.14.1 / 1.9.11 / 1.8.0 are the correct current versions, the guide version cards and "What's new" sections contain fabricated history and should be audited and corrected.
- If higher versions are available on PyPI but not accessible in this environment, the environment's network/mirror configuration should be investigated.

**Impact on merged PR content:** PRs #128 (google-adk Vol. 7, claimed against 2.1.0) and #129 (pydantic-ai Vol. 5, claimed against 1.104.0) contain deep-dive content that the previous sessions claimed to have verified against those versions. This run cannot independently verify that content because the installed packages are at lower versions. The PRs should be held for human review before merging if correctness of the deep-dive API details is critical.

**Note on pydantic-ai core symbols:** The following symbols from the comprehensive guide were verified present in the installed `pydantic-ai==1.31.0`: `Agent`, `RunContext`, `ModelRetry`, `UsageLimits`, `ModelMessage` — all PASS. The private path `pydantic_ai._agent_run.AgentRun` is absent; the public `AgentRun` is accessible via `pydantic_ai.AgentRun` (not verified in this run but likely present given the public API surface).
