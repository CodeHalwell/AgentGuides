### 2.2 Semantic Kernel — Carry-Forward Open Item Resolved

| | |
|---|---|
| **Version** | 1.41.3 (unchanged) |
| **Install environments** | `.routine-envs/main-py-0429-new` (Python 3.11.15); `/tmp/sk-test` (Python 3.12.3) |
| **Sources consulted** | PyPI wheel metadata (`semantic_kernel-1.41.3-py3-none-any.whl`): `Requires-Python: >=3.10`; `uv` resolution error log (verbose); standard `pip download` output; installed `semantic-kernel==1.41.3` source at `/tmp/sk-test/lib/python3.12/site-packages/` |

**Carry-forward resolution:** The previous open item stated: *"In Python 3.11.15, `uv pip install semantic-kernel` resolves 1.36.0 rather than 1.41.3, suggesting 1.37+ requires Python 3.12+"*. This hypothesis was incorrect. Investigation confirms:

- `uv pip install semantic-kernel` resolves to 1.36.0 on **both Python 3.11 and Python 3.12** without additional flags.
- The root cause is that semantic-kernel 1.37+ depends on `azure-ai-agents>=1.2.0b3`, a pre-release package. `uv` will not install packages that resolve to pre-release versions unless `--prerelease=allow` is passed.
- Standard `pip install semantic-kernel` correctly resolves 1.41.3 on both Python versions, because `pip`'s resolver accepts pre-release transitive dependencies when a stable package requires them.
- `Requires-Python: >=3.10` is confirmed correct from the 1.41.3 wheel metadata. Python 3.12 is **not** required.

**Guide change:** Added a callout to `semantic_kernel_comprehensive_python.md` §1 (Installation & Setup) noting that `uv` users must pass `--prerelease=allow` to get 1.37+, and that standard `pip install` is unaffected.

**Docs-vs-source discrepancy:** Importing `OpenAIChatCompletion` from `semantic_kernel.connectors.ai.open_ai` on semantic-kernel 1.41.3 + pydantic 2.11.10 triggers:

```
DeprecationWarning: The `__get_pydantic_core_schema__` method of the `BaseModel` class is deprecated. If you are calling `super().__get_pydantic_core_schema__` when overriding the method on a Pydantic model, consider using `handler(source)` instead. Deprecated in Pydantic V2.11 to be removed in V3.0.
```

This is an internal Pydantic V2.11 deprecation triggered by semantic-kernel's model classes. It does not affect the public API; guide examples using `OpenAIChatCompletion` remain correct. This will need to be fixed upstream in semantic-kernel before pydantic 3.0 ships. Flagged in Risk Register.
